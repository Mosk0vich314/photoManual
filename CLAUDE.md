# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A mobile-first React app that turns `Photo_Manual.pdf` (a 77-page Italian photography &
color-grading manual) into an interactive reference. The app's UI text and all content are
**in Italian** — keep it that way.

## Commands

```bash
npm install
npm run dev        # Vite dev server at http://localhost:5173
npm run build      # static build to dist/  (vite base is './', so it runs from any path)
npm run preview    # serve the production build
npm run extract    # python extract.py -> regenerates src/data/manual.json from the PDF

pip install -r requirements.txt   # PyMuPDF, needed only for extract.py
node shoot.mjs [route] [name]     # mobile screenshot via puppeteer-core + installed Chrome
```

There is no test runner or linter configured.

## The data pipeline is the heart of the project

Content is **not authored by hand** — it is extracted from the PDF, so most "content bugs"
are fixed in `extract.py`, not in JSX. Two data sources feed the app:

1. **`src/data/manual.json`** — generated by `extract.py`. Do not edit by hand; re-run
   `npm run extract`.
2. **`src/data/styleMeta.json`** — hand-curated metadata (per-style harmony, chromatic axes,
   color swatches, filter tags, and the scene→look table for the picker). Edit this directly.

`src/data/index.js` merges the two: it attaches `styleMeta.styles[num]` onto each style
chapter as `chapter.m`, splits chapters into `THEORY` / `STYLES` / `APPENDICES`, and provides
lookup helpers (`styleBySlug`, `chapterBySlug`) plus the full-text `search()`.

### How extract.py works (read before changing extraction)

PyMuPDF (`fitz`), not `pypdf` — pypdf drops spaces at font boundaries. The parser is driven
entirely by the LaTeX font names in the PDF, which encode structure reliably:

| Font (size)        | Meaning                              | Becomes |
|--------------------|--------------------------------------|---------|
| `SFBX1728` (17pt)  | chapter / style / appendix title     | new chapter |
| `SFBX1440` (14pt)  | section heading                      | `heading` block |
| `SFBX1095` "SCHEDA RAPIDA" | quick-card marker             | `scheda` block |
| `SFBX1200` (bold)  | inline parameter label               | run `k:"label"` |
| `SFTT*` (mono)     | slider value (e.g. `-15 / -25`)      | run `k:"value"` |
| `SFTI*` (italic)   | "Esempio visivo" lead-in / captions  | `example` / `caption` block |
| `SFRM*`            | body text                            | run `k:"text"` |
| size ≤ 8.5         | diagram tick labels / wheel degrees  | dropped as noise |

Other key behaviors: canonical titles come from the PDF's TOC (`get_toc()`), not the body
(the body titles are ALL CAPS); end-of-line soft hyphenation is rejoined; Scheda Rapida rows
are split by x-position (label column `x0≈71`, value column `x0≈200`).

### Output data model (`manual.json`)

```
{ meta, chapters: [ {
    id, slug, kind: "theory"|"style"|"appendix", num, title, film, director, page,
    blocks: [
      { type:"heading", text }
    | { type:"para", runs:[ {t, k:"text"|"label"|"value"|"emph"} ] }
    | { type:"example", text }      // "Esempio visivo: …"
    | { type:"caption", text }      // diagram caption
    | { type:"scheda", rows:[ {label, value} ] }
    ] } ] }
```

## App structure

- Routing: `HashRouter` (works on static hosting / `file://`). Routes live in `src/App.jsx`,
  which also renders the sticky topbar and the 5-tab bottom nav.
- Pages (`src/pages/`): `Home`, `Looks` (gallery + filters), `StyleDetail`, `Manuale` (TOC),
  `ChapterReader`, `Picker` (scene→look), `Search`.
- `src/components/Blocks.jsx` renders a chapter's block array. Its `widgets` prop injects an
  interactive component **after a heading with a given text** — this is how the color wheel
  (`HueWheel`) is placed into the *Teoria del Colore* chapter (key: `"La ruota e i gradi"`).
  Use the same mechanism to attach new interactive widgets to specific chapters/headings.
- `SchedaRapida.jsx` re-highlights numeric tokens inside the quick-card values.

## Conventions

- Two type families: a serif (`--font` default, the manual's voice) and a UI sans applied via
  the `.ui` class (nav, chips, labels). Colors are CSS variables in `src/styles.css`
  (`--gold` is the accent, `--value` is the slider-value badge color). Dark cinematic theme.
- Some styles have no parenthetical film in the TOC (e.g. Stile 2 "Bergman") → `film` is
  `null`. Components must fall back (usually to `m.harmony`); don't assume `film` exists.
- Style slugs are `stile-<num>-<title>`; link with `/look/<slug>`. Other chapters:
  `/capitolo/<slug>`. Helpers in `data/index.js`.
